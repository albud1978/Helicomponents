# Backlog

Короткие идеи на будущее, без реализации.

## Назначение и расположение
- Файл: `docs/backlog.md`
- Назначение: фиксация идей на будущее без обязательств по реализации.

## Формат записи (обязательные поля)
- Дата (YYYY-MM-DD)
- Краткое название
- Фаза реализации (например: "после стабилизации adaptive days")
- Контекст (1-2 строки)
- Следующий шаг (если нужен)
- Владелец (если определен)
